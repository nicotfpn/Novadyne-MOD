# Project Instructions

- Sempre que qualquer código, registro, recurso, configuração Gradle ou metadata do mod for alterado, atualize também `BLUEPRINT.md` para refletir o estado atual do projeto.
- Mantenha o `BLUEPRINT.md` como referência rápida da estrutura, classe principal, registros, recursos, configurações e observações do projeto.
- Para mudanças em itens, blocos, entidades, creative tabs, assets, data packs, receitas, loot tables, tags ou dependências, registre a alteração correspondente no blueprint no mesmo trabalho.
